NONPROFIT GRANT WRITING BEST PRACTICES – COMPREHENSIVE GUIDE  
==================================================================  

INTRODUCTION  
------------  
Grant writing is a strategic, repeatable process that aligns your mission with funder priorities.  
A successful proposal tells a clear story: why the problem matters, what you will do, how you will measure success, and what it will cost.  
This guide walks you through every stage – from funder research to post‑award reporting – with concrete examples and practical checklists.  

SECTION 1 – RESEARCHING FUNDERS  
-------------------------------  
1.1 DEFINE YOUR FUNDING NEEDS  
- List program areas, geographic focus, population served, and required dollar range.  
- Example: “Youth STEM after‑school program, urban Midwest, $75,000‑$150,000 per year.”  

1.2 BUILD A FUNDER DATABASE  
- Use Foundation Directory Online, GrantStation, GuideStar, and local community foundation listings.  
- Capture: funder name, mission, giving history, average grant size, deadline, contact person, and application format.  

1.3 PRIORITIZE FUNDERS WITH FIT SCORE  
- Score each funder 1‑5 on: mission alignment, geographic match, funding history, likelihood of renewal, and application burden.  
- Target funders scoring ≥12/20 for first‑round submissions.  

1.4 ANALYZE PAST GRANTS  
- Download 3‑5 recent awards from each target funder.  
- Note language patterns, required attachments, and evaluation expectations.  

1.5 CREATE A FUNDER PROFILE TEMPLATE (CHECKLIST)  
[ ] Funder name & contact  
[ ] Mission statement  
[ ] Giving priorities (last 3 years)  
[ ] Average grant size & range  
[ ] Application deadline & format  
[ ] Required attachments (budget, logic model, letters of support)  
[ ] Review criteria (explicit or inferred)  
[ ] Decision timeline  
[ ] Renewal history  
[ ] Internal champion (board member, staff)  

SECTION 2 – CULTIVATING RELATIONSHIPS  
------------------------------------  
2.1 INITIAL OUTREACH  
- Send a brief, personalized email (≤150 words) referencing a specific grant or program.  
- Attach a one‑page “concept note” summarizing the problem, solution, and ask.  

2.2 REQUEST A MEETING  
- Ask for a 20‑minute call to learn about the funder’s current priorities.  
- Prepare 3‑4 thoughtful questions (e.g., “What outcomes are you most excited to see in the next funding cycle?”).  

2.3 MAINTAIN A CONTACT LOG (CHECKLIST)  
[ ] Date of contact  
[ ] Method (email, phone, meeting)  
[ ] Summary of discussion  
[ ] Action items & deadlines  
[ ] Follow‑up date  

2.4 LEVERAGE BOARD & VOLUNTEER NETWORKS  
- Map board members’ connections to target funders.  
- Assign a “relationship owner” for each high‑priority funder.  

SECTION 3 – CRAFTING A COMPELLING NEEDS STATEMENT  
---------------------------------------------------  
3.1 START WITH DATA, NOT ANECDOTE  
- Use recent, reputable sources (Census, CDC, local health department, peer‑reviewed studies).  
- Example: “In County X, 27% of children ages 5‑12 live below the federal poverty line (U.S. Census 2023).”  

3.2 QUANTIFY THE GAP  
- Show the difference between current services and demand.  
- Example: “Only 12% of eligible youth have access to after‑school STEM programming (State Dept. of Education 2022).”  

3.3 HUMANIZE WITH A STORY  
- Insert a brief, anonymized vignette (2‑3 sentences) that illustrates the data.  
- Example: “Maria, a 10‑year‑old in Eastside, has never used a microscope because her school lacks a lab.”  

3.4 CONNECT TO FUNDER PRIORITIES  
- Explicitly map each data point to a funder’s stated focus area.  
- Use the funder’s own language when possible.  

3.5 NEEDS STATEMENT CHECKLIST  
[ ] Current, citation‑ready statistics  
[ ] Clear gap between need and existing services  
[ ] One compelling human story  
[ ] Direct alignment with funder priorities  
[ ] No jargon; plain language  
[ ] Length ≤ 500 words (or funder limit)  

SECTION 4 – DEFINING GOALS, OBJECTIVES, AND OUTCOMES  
----------------------------------------------------  
4.1 GOAL (Broad, aspirational)  
- “Increase STEM engagement among low‑income middle‑school students in County X.”  

4.2 SMART OBJECTIVES (Specific, Measurable, Achievable, Relevant, Time‑bound)  
- Objective 1: Enroll 120 students in the after‑school STEM club by 12/31/2026.  
- Objective 2: Achieve a 20% increase in participants’ science‑fair project scores by 06/30/2027.  

4.3 OUTCOME INDICATORS  
- Short‑term: Attendance rates, pre‑/post‑survey knowledge gains.  
- Medium‑term: School‑reported STEM course enrollment.  
- Long‑term: College STEM major declaration (tracked via alumni survey).  

4.4 OBJECTIVES CHECKLIST  
[ ] Each objective is SMART  
[ ] Objectives directly address needs statement  
[ ] Indicators are quantifiable and data‑source identified  
[ ] Baseline data collected or plan for collection  
[ ] Targets are realistic given resources  

SECTION 5 – BUILDING A LOGIC MODEL  
-----------------------------------  
5.1 COMPONENTS  
- Inputs → Activities → Outputs → Short‑term Outcomes → Long‑term Impact  

5.2 EXAMPLE LOGIC MODEL (TEXT VERSION)  
Inputs: $150,000 grant, 2 FTE staff, 5 volunteer mentors, lab equipment  
Activities: Recruit students, deliver 30 weekly 2‑hour sessions, host quarterly family STEM nights  
Outputs: 120 students served, 30 sessions delivered, 4 family events held  
Short‑term Outcomes: 85% attendance, 30% knowledge gain on post‑test  
Medium‑term Outcomes: 40% increase in school STEM elective enrollment  
Long‑term Impact: 15% rise in college STEM majors from target population  

5.3 LOGIC MODEL CHECKLIST  
[ ] All five columns populated  
[ ] Clear causal arrows (if‑then statements)  
[ ] Each output linked to at least one outcome  
[ ] Indicators assigned to each outcome level  
[ ] Model fits on one page (or funder‑specified format)  

SECTION 6 – DESIGNING THE PROGRAM PLAN  
--------------------------------------  
6.1 DETAILED WORK PLAN (GANTT‑STYLE)  
- List major tasks, responsible staff, start/end dates, milestones.  
- Example row: “Curriculum development – Lead: Program Director – 01/01/2026‑03/31/2026 – Milestone: 12 lesson plans approved.”  

6.2 STAFFING PLAN  
- Include FTE, qualifications, and % time allocated to grant.  
- Example: “STEM Coordinator – 1.0 FTE – Master’s in Education – 100% grant time.”  

6.3 PARTNERSHIP AGREEMENTS  
- Secure MOUs with schools, libraries, community centers.  
- Attach letters of support (≤1 page each).  

6.4 RISK MANAGEMENT MATRIX (CHECKLIST)  
[ ] Identify top 5 risks (e.g., staff turnover, low recruitment)  
[ ] Rate likelihood (High/Med/Low) & impact (High/Med/Low)  
[ ] Define mitigation strategy for each  
[ ] Assign owner & monitoring frequency  

SECTION 7 – EVALUATION PLAN  
---------------------------  
7.1 EVALUATION QUESTIONS  
- Process: “Was the program delivered as planned?”  
- Outcome: “Did participants’ STEM knowledge increase?”  
- Impact: “Are alumni pursuing STEM pathways?”  

7.2 DATA COLLECTION METHODS  
- Surveys (pre/post), attendance logs, school records, focus groups.  
- Use validated instruments where possible (e.g., STEM Attitude Survey).  

7.3 ANALYSIS PLAN  
- Descriptive stats for outputs, paired t‑tests for knowledge gains, regression for medium‑term outcomes.  

7.4 REPORTING SCHEDULE  
- Quarterly internal dashboards, semi‑annual funder reports, final impact report.  

7.5 EVALUATION CHECKLIST  
[ ] Evaluation questions aligned with logic model  
[ ] Data sources identified & access secured  
[ ] Instruments piloted or validated  
[ ] Analysis plan feasible with staff expertise  
[ ] Timeline matches funder reporting deadlines  

SECTION 8 – BUDGET DEVELOPMENT  
------------------------------  
8.1 BUDGET CATEGORIES (COMMON)  
- Personnel (salaries, fringe)  
- Fringe Benefits  
- Travel  
- Equipment (> $5,000)  
- Supplies  
- Contractual/Consultants  
- Indirect Costs (if allowed)  
- Other (e.g., participant stipends)  

8.2 BUILDING THE BUDGET SPREADSHEET  
- Use funder’s template or a standard 3‑column layout: Year 1, Year 2, Total.  
- Show calculations (e.g., “STEM Coordinator 1.0 FTE × $55,000 × 1.0 = $55,000”).  

8.3 COST‑SHARE & IN‑KIND  
- Document any matching funds or donated space/equipment.  
- Value in‑kind at fair market rate with supporting documentation.  

8.4 BUDGET NARRATIVE (JUSTIFICATION)  
- Explain each line item in 2‑3 sentences.  
- Link costs to activities in the work plan.  
- Example: “Travel $3,200 – mileage for Program Director to visit 4 partner schools (800 miles × $0.40).”  

8.5 BUDGET CHECKLIST  
[ ] All line items match work plan activities  
[ ] Calculations shown & error‑free  
[ ] Fringe rate matches organizational policy  
[ ] Indirect cost rate approved (or waived)  
[ ] Total does not exceed funder ceiling  
[ ] Cost‑share documented with letters  
[ ] Narrative references logic model outputs  

SECTION 9 – WRITING THE PROPOSAL NARRATIVE  
------------------------------------------  
9.1 FOLLOW FUNDER’S OUTLINE EXACTLY  
- Use the same headings, order, and word limits.  

9.2 STRUCTURE EACH SECTION  
- Opening hook (1‑2 sentences) → Evidence → Solution → Impact → Sustainability.  

9.3 USE ACTIVE VOICE & CONCRETE VERBS  
- “We will recruit” not “Recruitment will be conducted.”  

9.4 INTEGRATE DATA VISUALS  
- Insert a simple table or chart (e.g., enrollment targets) if page limits allow.  

9.5 STORYTELEMENT: “THEORY OF CHANGE” PARAGRAPH  
- Summarize logic model in narrative form for reviewers who skim.  

9.6 NARRATIVE CHECKLIST  
[ ] All funder‑required sections present  
[ ] Word counts respected  
[ ] No orphan acronyms (define first use)  
[ ] Consistent terminology (e.g., “participants” not “youth” then “students”)  
[ ] Proofread for grammar, spelling, formatting  

SECTION 10 – REVIEW, EDIT, AND INTERNAL APPROVAL  
------------------------------------------------  
10.1 PEER REVIEW PANEL (3‑5 reviewers)  
- Assign each reviewer a focus: needs, logic model, budget, evaluation, compliance.  

10.2 RED‑TEAM CRITIQUE  
- Simulate a reviewer’s score sheet; note any “red flags.”  

10.3 REVISION CYCLE  
- Round 1: Content edits (major gaps).  
- Round 2: Language polish (clarity, tone).  
- Round 3: Final compliance check (attachments, signatures).  

10.4 FINAL COMPLIANCE CHECKLIST  
[ ] All required forms signed (authorizing official, fiscal agent)  
[ ] Appendices labeled & ordered per funder guide  
[ ] Page limits obeyed (including appendices)  
[ ] File format & naming convention correct (PDF, “OrgName_GrantName_2026.pdf”)  
[ ] Submission portal tested (login, upload size)  

SECTION 11 – SUBMISSION & FOLLOW‑UP  
-----------------------------------  
11.1 SUBMIT EARLY  
- Aim for 48 hours before deadline to avoid technical issues.  

11.2 CONFIRM RECEIPT  
- Save confirmation email & screenshot of portal status.  

11.3 POST‑SUBMISSION LOG (CHECKLIST)  
[ ] Date & time submitted  
[ ] Confirmation reference number  
[ ] Contact person at funder (if known)  
[ ] Internal debrief date (within 1 week)  

SECTION 12 – POST‑AWARD MANAGEMENT  
----------------------------------  
12.1 AWARD LETTER REVIEW  
- Note special conditions, reporting dates, payment schedule, allowable cost changes.  

12.2 SET UP GRANT ACCOUNTING  
- Create separate fund code in accounting system.  
- Load budget vs. actual tracking template.  

12.3 KICK‑OFF MEETING (INTERNAL)  
- Review deliverables, assign task owners, set calendar reminders for reports.  

12.4 COMPLIANCE MONITORING (MONTHLY)  
- Verify expenditures against budget.  
- Track progress on logic‑model outputs.  

12.5 AMENDMENT PROCESS  
- Document any scope or budget changes; submit formal amendment request per funder policy.  

SECTION 13 – REPORTING TO FUNDERS  
---------------------------------  
13.1 PROGRESS REPORTS (QUARTERLY/SEMI‑ANNUAL)  
- Narrative: activities completed, deviations, successes, challenges.  
- Quantitative: updated logic‑model dashboard (outputs & outcomes).  
- Financial: budget vs. actual, variance explanation >10%.  

13.2 FINAL REPORT  
- Summarize all outcomes, impact evidence, lessons learned.  
- Include sustainability plan (next funding, earned income, policy change).  

13.3 REPORTING CHECKLIST  
[ ] Narrative follows funder template  
[ ] All required metrics reported with data sources  
[ ] Financial statements reconciled to accounting system  
[ ] Attachments (photos, testimonials, evaluation instruments) included  
[ ] Submitted by deadline, receipt confirmed  

SECTION 14 – SUSTAINABILITY & LEVERAGING THE GRANT  
---------------------------------------------------  
14.1 DIVERSIFY REVENUE  
- Map current funding mix; set target % for each source (grants, contracts, individual giving, earned income).  

14.2 BUILD EVIDENCE BASE  
- Publish evaluation findings in newsletters, briefs, or peer‑reviewed journals.  

14.3 CULTIVATE FUNDER RELATIONSHIPS POST‑AWARD  
- Invite funder staff to site visits, share impact stories, ask for introductions.  

14.4 SUSTAINABILITY CHECKLIST  
[ ] Written sustainability plan (1‑2 pages) attached to final report  
[ ] At least two new funding prospects identified per grant cycle  
[ ] Board briefed on grant outcomes & future needs  
[ ] Data package ready for next proposal (updated logic model, outcomes)
